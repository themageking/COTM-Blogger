Ao construir um blog com Next.js, uma das decisões de arquitetura mais importantes é como suas páginas serão entregues ao usuário. Essa escolha impacta diretamente a velocidade, segurança e o potencial de ranqueamento do seu site.

As duas principais abordagens de pré-renderização no Next.js são SSG e SSR. Vamos entender a diferença.

### Server-Side Rendering (SSR)

Com SSR, a página HTML é gerada no servidor **a cada nova requisição**. Isso é perfeito para conteúdo altamente dinâmico, que muda a todo instante, como o feed de uma rede social ou o placar de um jogo ao vivo. A página está sempre atualizada, mas exige um servidor rodando constantemente.

### Static Site Generation (SSG)

Com SSG, as páginas são geradas **uma única vez**, durante o processo de *build* (construção do site). O resultado é um conjunto de arquivos HTML, CSS e JavaScript puros, prontos para serem servidos instantaneamente.

## Por que SSG é a Escolha Perfeita para Blogs?

Para um blog, onde o conteúdo é publicado e raramente alterado depois, a abordagem SSG é a estratégia vencedora por vários motivos:

* **Velocidade Absurda:** Como as páginas já existem como arquivos estáticos, elas carregam de forma quase instantânea para o usuário.
* **Segurança Reforçada:** Sem um processo de geração de página a cada visita, a superfície de ataque a um servidor é drasticamente reduzida.
* **SEO Imbatível:** Motores de busca como o Google adoram sites estáticos. O conteúdo está todo no HTML, tornando a indexação rápida, fácil e eficiente.
* **Hospedagem Simplificada e Barata:** Servir arquivos estáticos é barato e pode ser feito por qualquer serviço de hospedagem ou CDN, com altíssima performance.
